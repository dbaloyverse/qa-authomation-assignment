Task Board - Test Scenarios

QA Automation Live Coding Assessment

Instructions Scenarios Open Task Board App

Assessment Instructions

Implement automated tests for the Task Board application using your preferred test framework (Selenium, Cypress, Playwright, etc.).

Your solution should demonstrate:

  • Page Object Model or similar design pattern
  • Proper wait strategies and synchronization
  • Clean, maintainable test code
  • Handling of dynamic elements and overlays
  • Appropriate assertions and verifications

Time: Choose 3-5 scenarios to implement based on your available time.

Scenario 1: Create a New Task

Easy 
@smoke @create-task
Feature: Task Creation

  Scenario: Successfully create a new task with all fields
    Given the Task Board page is loaded
    When I enter "Implement login feature" in the title field
    And I enter "Add OAuth2 authentication flow" in the description field
    And I select "High" priority
    And I click the Add Task button
    Then a success toast should appear with message "Task created successfully"
    And the task "Implement login feature" should appear in the To Do column
    And the task should display "High" priority badge

  Scenario: Create task with only required fields
    Given the Task Board page is loaded
    When I enter "Quick task" in the title field
    And I click the Add Task button
    Then the task "Quick task" should appear in the To Do column

Scenario 2: Form Validation

Easy 
@validation
Feature: Form Validation

  Scenario: Validation error when submitting empty title
    Given the Task Board page is loaded
    When I click the Add Task button without entering a title
    Then the title input should display an error state
    And the error message "Title is required" should be visible
    And no new task should be created

  Scenario: Error state clears after entering valid title
    Given the Task Board page is loaded
    And I have triggered the title validation error
    When I enter "Valid task title" in the title field
    And I click the Add Task button
    Then the title input should not display an error state
    And the task should be created successfully

Scenario 3: Delete a Task

Easy 
@delete-task
Feature: Task Deletion

  Scenario: Delete a task from the board
    Given the Task Board page is loaded
    And a task "Create API endpoints" exists in the To Do column
    When I click the menu button on task "Create API endpoints"
    And I click the Delete button
    Then a success toast should appear with message "Task deleted successfully"
    And the task "Create API endpoints" should not exist on the board

Scenario 4: Move Task Through Columns

Medium 
@move-task @workflow
Feature: Task Movement

  Scenario: Move task from To Do to In Progress
    Given the Task Board page is loaded
    And a task "Create API endpoints" exists in the To Do column
    When I click the menu button on task "Create API endpoints"
    And I click "Move Forward"
    Then a success toast should appear with message "Task moved successfully"
    And the task should appear in the In Progress column
    And the task should NOT exist in the To Do column

  Scenario: Move task through all columns to Done
    Given the Task Board page is loaded
    And a task exists in the To Do column
    When I move the task forward twice
    Then the task should appear in the Done column

  Scenario: Move Forward not available for Done tasks
    Given the Task Board page is loaded
    And a task exists in the Done column
    When I click the menu button on that task
    Then the "Move Forward" button should NOT be visible
    And only the Delete button should be available

Scenario 5: Search and Navigate to Task

Medium 
@search
Feature: Task Search

  Scenario: Search for task by title

    And multiple tasks exist on the board
    When I type "authentication" in the search field
    Then the search results dropdown should appear
    And the results should contain "Implement user authentication"

  Scenario: Navigate to task from search results
    Given the Task Board page is loaded
    When I type "database" in the search field
    And I click on the result "Design database schema"
    Then the page should scroll to the task
    And the task should be visually highlighted
    And the search input should be cleared

  Scenario: No results for non-matching search

    When I type "xyznonexistent" in the search field
    Then no search results dropdown should appear

Scenario 6: Cookie Consent Banner

Medium 
@cookie-banner
Feature: Cookie Consent Banner

  Scenario: Cookie banner appears on first visit
    Given I have cleared browser cookies for the application

    Then the cookie consent banner should appear

  Scenario: Accepting cookies dismisses banner
    Given the cookie consent banner is visible
    When I click the Accept button
    Then the cookie banner should disappear
    And the banner should not reappear on page refresh

  Scenario: Declining cookies dismisses banner

Scenario 7: Complete Task Workflow

Hard 
@e2e @workflow
Feature: Complete Task Lifecycle

  Scenario: Create and move task through all columns then delete
    Given the Task Board page is loaded

    # Create new task
    When I create a new task with title "E2E Test Task" and priority "High"
    Then the task should appear in the To Do column

    # Move to In Progress
    When I move task "E2E Test Task" forward
    Then the task should be in the In Progress column

    # Move to Done
    When I move task "E2E Test Task" forward
    Then the task should be in the Done column

    # Search and verify
    When I search for "E2E Test"
    Then the task should appear in search results

    # Delete
    When I delete the task "E2E Test Task"
    Then the task should no longer exist on the board
    And searching for "E2E Test" should return no results

Scenario 8: Verify Loading States

Hard 
@loading
Feature: Loading State Verification

  Scenario: Loading overlay appears during task creation
    Given the Task Board page is loaded
    When I submit a new task
    Then a loading overlay should appear
    And I should not be able to interact with other elements
    When the loading completes
    Then the overlay should disappear
    And the task should be created

  Scenario: Loading overlay appears during task movement
    Given the Task Board page is loaded
    And a task exists in the To Do column
    When I move the task forward
    Then a loading overlay should appear briefly
    And after loading completes the task should be in the new column

Scenario 9: Toast Notification Verification

Hard 
@toast
Feature: Toast Notifications

  Scenario: Verify toast appears with correct message
    Given the Task Board page is loaded
    When I create a new task
    Then a success toast should appear
    And the toast should contain message "Task created successfully"
    And the toast should have a close button

  Scenario: Toast auto-dismisses
    Given a toast notification is visible
    When I wait without interacting
    Then the toast should automatically disappear

  Scenario: Manually dismiss toast
    Given a toast notification is visible
    When I click the close button on the toast
    Then the toast should immediately disappear

Scenario 10: Column Task Count Verification

Hard 
@counts @verification
Feature: Column Count Verification

  Scenario: Task counts update when creating tasks
    Given the Task Board page is loaded
    And I note the current To Do column count
    When I create a new task
    Then the To Do column count should increase by 1

  Scenario: Task counts update when moving tasks
    Given the Task Board page is loaded
    And I note the counts for To Do and In Progress columns
    When I move a task from To Do to In Progress
    Then the To Do count should decrease by 1
    And the In Progress count should increase by 1

  Scenario: Task counts update when deleting tasks
    Given the Task Board page is loaded
    And I note the current Done column count
    When I delete a task from the Done column
    Then the Done column count should decrease by 1
