Assessment Overview

This page contains Gherkin test scenarios for the Task Board application. Candidates should implement automated tests using their preferred framework (Selenium, Cypress, Playwright, etc.) to cover these scenarios.

The application is intentionally designed with common automation challenges. Candidates must identify and handle these challenges appropriately.

Automation Challenges to Handle

Cookie Consent Banner

Appears after 500ms delay, blocks bottom elements (z-index: 9999)

Loading Overlay

Random delays (300-2000ms), blocks all interactions

Dynamic Menu

Uses @if directive - completely removed from DOM when closed

Toast Notifications

Auto-dismiss after 3 seconds, must verify before disappearing

Fixed Header

60px height, may cause ElementClickInterceptedException

Debounced Search

300ms debounce + 200-400ms API delay before results appear

Stale Elements

Task IDs regenerate on move, causing stale element references

Button State Changes

Submit button disabled during form submission

Basic Scenarios

Scenario 1: Create a New Task

Easy
@smoke @create-task Feature: Task Creation Background: Given the Task Board page is loaded And the cookie consent banner is dismissed Scenario: Successfully create a new task with all fields When I enter "Implement login feature" in the title field And I enter "Add OAuth2 authentication flow" in the description field And I select "High" priority And I click the Add Task button Then a success toast should appear with message "Task created successfully" And the task "Implement login feature" should appear in the To Do column And the task should display "High" priority badge Scenario: Create task with only required fields When I enter "Quick task" in the title field And I click the Add Task button Then a success toast should appear with message "Task created successfully" And the task "Quick task" should appear in the To Do column

Automation Challenges:

  • Cookie banner must be dismissed before interacting with form
  • Loading overlay appears during task creation (800-1200ms)
  • Submit button changes to "Saving..." and becomes disabled during submission
  • Toast notification auto-dismisses after 3 seconds - capture message quickly

Scenario 2: Form Validation

Easy
@validation @negative-test Feature: Form Validation Background: Given the Task Board page is loaded And the cookie consent banner is dismissed Scenario: Validation error when submitting empty title When I click the Add Task button without entering a title Then the title input should display an error state And the error message "Title is required" should be visible And no new task should be created Scenario: Error state clears after entering valid title Given I have triggered the title validation error When I enter "Valid task title" in the title field And I click the Add Task button Then the title input should not display an error state And the task should be created successfully

Automation Challenges:

  • Verify CSS class "error" is added/removed dynamically
  • Error message element appears/disappears with @if directive
  • Negative test - must verify task was NOT created

Scenario 3: Delete a Task

Easy
@delete-task Feature: Task Deletion Background: Given the Task Board page is loaded And the cookie consent banner is dismissed And the initial tasks have loaded Scenario: Delete a task from To Do column Given a task "Create API endpoints" exists in the To Do column When I click the menu button on task "Create API endpoints" And I click the Delete button Then the loading overlay should appear And a success toast should appear with message "Task deleted successfully" And the task "Create API endpoints" should not exist on the board Scenario: Delete a task from Done column Given a task "Setup project repository" exists in the Done column When I delete the task "Setup project repository" Then the task should be removed from the board

Automation Challenges:

  • Menu appears only after clicking the menu button (â‹®)
  • Menu uses @if - Delete button is not in DOM until menu is open
  • Must wait for menu to appear before clicking Delete
  • Loading overlay blocks during deletion (300-500ms)

Intermediate Scenarios

Scenario 4: Move Task Through Columns

Medium
@move-task @workflow Feature: Task Movement Background: Given the Task Board page is loaded And the cookie consent banner is dismissed Scenario: Move task from To Do to In Progress Given a task "Create API endpoints" exists in the To Do column When I click the menu button on task "Create API endpoints" And I click "Move Forward" Then the loading overlay should appear briefly And a success toast should appear with message "Task moved successfully" And the task should appear in the In Progress column And the task should NOT exist in the To Do column Scenario: Move task from In Progress to Done Given a task "Implement user authentication" exists in the In Progress column When I move the task forward Then the task should appear in the Done column And the In Progress column count should decrease by 1 And the Done column count should increase by 1 Scenario: Move Forward button not available for Done tasks Given a task exists in the Done column When I click the menu button on that task Then the "Move Forward" button should NOT be visible And only the Delete button should be available

Automation Challenges:

  • Stale Element Reference: Task ID changes after move - must re-locate element
  • Menu closes after clicking action - re-open to interact again
  • Verify element is removed from source column (not just hidden)
  • Column count updates dynamically - verify count changes

Scenario 5: Search and Navigate to Task

Medium
@search @navigation Feature: Task Search Background: Given the Task Board page is loaded And the cookie consent banner is dismissed And multiple tasks exist on the board Scenario: Search for task by title When I type "authentication" in the search field Then the search results dropdown should appear And the results should contain "Implement user authentication" Scenario: Navigate to task from search results When I type "database" in the search field And I wait for search results to appear And I click on the result "Design database schema" Then the page should scroll to the task And the task should be visually highlighted And the search input should be cleared And the search results should close Scenario: No results for non-matching search When I type "xyznonexistent" in the search field And I wait for the search to complete Then no search results dropdown should appear

Automation Challenges:

  • Debounce delay: Results don't appear immediately - 300ms debounce + 200-400ms API delay
  • Search results dropdown uses @if - not in DOM until results exist
  • Must verify scroll position changed after clicking result
  • Highlight animation lasts ~2 seconds then fades

Scenario 6: Cookie Consent Banner

Medium
@cookie-banner @overlay Feature: Cookie Consent Banner Scenario: Cookie banner appears on first visit Given I have cleared browser cookies for the application When I navigate to the Task Board page Then the page should load without the cookie banner initially And after 500 milliseconds the cookie banner should appear And the banner should contain Accept and Decline buttons Scenario: Accepting cookies dismisses banner Given the cookie consent banner is visible When I click the Accept button Then the cookie banner should disappear And the banner should not reappear on page refresh Scenario: Cookie banner blocks bottom elements Given the cookie consent banner is visible And a task card is positioned near the bottom of the viewport When I try to click on that task card Then the click may be intercepted by the cookie banner # Candidate should handle this by dismissing banner first

Automation Challenges:

  • Banner appears with 500ms delay - not immediately on page load
  • Banner has z-index: 9999 - blocks clicks on elements behind it
  • Must wait for banner to appear before dismissing
  • Consider adding banner dismissal to test setup/hooks

Advanced Scenarios

Scenario 7: Complete Task Workflow

Hard
@e2e @workflow @regression Feature: Complete Task Lifecycle Background: Given the Task Board page is loaded And the cookie consent banner is dismissed Scenario: Create and move task through all columns # Step 1: Create new task When I create a new task with title "E2E Test Task" and priority "High" Then the task should appear in the To Do column And the To Do column count should increase # Step 2: Move to In Progress When I move task "E2E Test Task" forward Then the task should be in the In Progress column And the To Do count should decrease And the In Progress count should increase # Step 3: Move to Done When I move task "E2E Test Task" forward Then the task should be in the Done column # Step 4: Verify and delete When I search for "E2E Test" Then the task should appear in search results When I delete the task Then the task should no longer exist on the board And searching for "E2E Test" should return no results

Automation Challenges:

  • Multiple stale element situations - task ID changes on each move
  • Must re-locate task after each move operation
  • Track column counts across multiple operations
  • Handle multiple loading overlays sequentially
  • Verify both positive and negative states (exists / doesn't exist)

Scenario 8: Loading State Handling

Hard
@loading @synchronization Feature: Loading State Management Scenario: Initial page load with loading overlay When I navigate to the Task Board page Then a loading overlay should be visible And no tasks should be visible yet When the loading overlay disappears Then all pre-existing tasks should be visible And the board should have tasks in multiple columns Scenario: Cannot interact during loading Given I am creating a new task When the loading overlay is visible Then I should not be able to click on task cards And I should not be able to submit another task When the loading overlay disappears Then all interactions should be possible again Scenario: Handle random loading delays Given I perform multiple task operations Then each operation should complete regardless of delay duration And my tests should not use hard-coded sleep values # Tests should use explicit waits for loading overlay to disappear

Automation Challenges:

  • Loading overlay has random duration - hard-coded waits will fail
  • Must implement explicit wait for loading overlay to disappear
  • Initial load: 500-800ms, Create: 800-1200ms, Move: 400-700ms, Delete: 300-500ms
  • Loading overlay has z-index: 5000 - blocks all clicks

Scenario 9: Toast Notification Verification

Hard
@toast @timing Feature: Toast Notifications Scenario: Verify toast appears and auto-dismisses When I create a new task Then a success toast should appear within 2 seconds And the toast should contain message "Task created successfully" And the toast should have a close button When I wait 3 seconds without dismissing Then the toast should automatically disappear Scenario: Multiple toasts stack correctly When I quickly create two tasks in succession Then two toast notifications should be visible And they should be stacked vertically And each should dismiss after 3 seconds Scenario: Manually dismiss toast When I perform an action that shows a toast And I click the close button on the toast Then the toast should immediately disappear

Automation Challenges:

  • Toast exists for only 3 seconds - must capture quickly
  • Toast animates in/out - may need to wait for animation
  • Multiple toasts can exist simultaneously
  • Consider using explicit wait with custom condition for toast text

Scenario 10: Data-Driven Task Creation

Hard
@data-driven @parameterized Feature: Data-Driven Tests Scenario Outline: Create tasks with different priorities Given the Task Board page is loaded And the cookie consent banner is dismissed When I create a task with title "<title>" and priority "<priority>" Then the task should appear in the To Do column And the task should display the "<priority>" priority badge And the task card should have the "<css_class>" border style Examples: | title | priority | css_class | | Low priority task | Low | priority-low | | Medium task | Medium | priority-medium | | Urgent task | High | priority-high | Scenario Outline: Search with various queries When I search for "<query>" Then I should see "<expected_count>" results Examples: | query | expected_count | | project | 1 | | database | 1 | | implementation | 0 |

Automation Challenges:

  • Implement proper test data management
  • Clean up created tasks after each test iteration
  • Verify CSS classes dynamically based on priority
  • Handle test isolation - each iteration should be independent
